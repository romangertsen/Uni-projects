# Применение машинного обучения в задаче оптимизации инвестиционного портфеля

Проект посвящён исследованию методов Financial Machine Learning для фильтрации инвестиционных сигналов и управления размером позиции при торговле ETF.

Работа основана на методологии Marcos López de Prado из книги *Advances in Financial Machine Learning* и включает полный цикл построения инвестиционной стратегии: сбор данных, построение признаков, событийную выборку наблюдений, разметку данных, обучение моделей машинного обучения и тестирование инвестиционных результатов.

---

# Описание работы

Исследование состоит из следующих этапов:

1. Сбор данных по ETF и макроэкономическим индикаторам.
2. Построение технических и рыночных признаков.
3. Стационаризация финансовых рядов методом Fractional Differentiation.
4. Формирование событий с помощью CUSUM-фильтра.
5. Разметка событий методом Triple Barrier Method.
6. Построение meta-labeling модели.
7. Взвешивание пересекающихся наблюдений.
8. Отбор признаков.
9. Обучение моделей машинного обучения.
10. Walk-forward backtesting.
11. Анализ влияния управления размером позиции.

# Построение признаков

Для каждого ETF рассчитываются несколько групп признаков.

## Momentum-признаки

Доходности за окна:

- 1 день
- 5 дней
- 21 день
- 63 дня
- 252 дня

Признаки отражают наличие краткосрочного и долгосрочного тренда.

## Volatility-признаки

Используются:

- реализованная волатильность
- отношение краткосрочной и долгосрочной волатильности

Признаки помогают определить текущий рыночный режим.

## Drawdown-признаки

Для разных окон рассчитывается максимальная просадка относительно локального максимума.

Признаки отражают силу недавнего падения рынка.

## Volume-признаки

Используются:

- отношение объёма к скользящему среднему
- изменение объёма
- волатильность объёма

# Fractional Differentiation

Финансовые ценовые ряды обычно нестационарны.

Обычное дифференцирование делает ряд стационарным, но уничтожает значительную часть долгосрочной информации.

Поэтому используется Fractional Differentiation — дробное дифференцирование.

Для каждого ETF:

1. берутся логарифмы цен
2. подбирается минимальный параметр `d`
3. выполняется ADF-тест
4. выбирается минимальное значение `d`, при котором ряд становится стационарным

Такой подход позволяет сохранить часть долгосрочной памяти рынка и одновременно уменьшить нестационарность.

---

# CUSUM-фильтр

После построения признаков применяется симметричный CUSUM-фильтр.

CUSUM (Cumulative Sum Filter) используется для событийной выборки наблюдений.

Вместо использования каждого торгового дня алгоритм отслеживает накопленное изменение ряда.

Когда накопленное отклонение превышает заданный порог:

- фиксируется событие
- событие добавляется в обучающий набор
- накопленная сумма обнуляется

Периоды без значимых движений игнорируются.

Это позволяет:

- уменьшить влияние шума
- сократить размер выборки
- оставить только информативные рыночные события

Порог выбирается как одно стандартное отклонение изменения дробно-дифференцированного ряда.

---

# Triple Barrier Method

Для каждого события строятся три барьера:

- Take Profit
- Stop Loss
- Вертикальный временной барьер

В проекте используются параметры:

- Take Profit = 1 волатильность
- Stop Loss = 2 волатильности
- Вертикальный барьер = 5 торговых дней

Событие получает метку:

- `1` — если сигнал оказался прибыльным
- `0` — если сигнал оказался убыточным или нейтральным

---

# Meta-Labeling

Базовая стратегия использует momentum за 63 дня.

Если momentum положителен:

- открывается длинная позиция

Если momentum отрицателен:

- открывается короткая позиция

Модель машинного обучения не прогнозирует направление рынка.

Она прогнозирует успешность уже существующего momentum-сигнала.

Таким образом модель решает:

- принимать сигнал
- игнорировать сигнал

# Взвешивание наблюдений

События могут пересекаться во времени.

Несколько сделок могут использовать одни и те же рыночные движения.

Для уменьшения переобучения используются:

- sample uniqueness
- return attribution

Сильно пересекающиеся события получают меньший вес при обучении модели.

---

# Отбор признаков

Используется Single Feature Importance (SFI).

Для каждого признака:

1. модель обучается только на этом признаке
2. качество оценивается через PurgedKFold
3. вычисляется F1 Macro

В итоговую модель попадают только наиболее информативные признаки.

# PurgedKFold и Embargo

Обычная кросс-валидация плохо подходит для финансовых данных из-за пересечения событий во времени.

Используется специальная схема валидации.

## PurgedKFold

Удаляет из обучения наблюдения, пересекающиеся с тестовым периодом.

## Embargo

Дополнительно удаляет небольшой участок данных после тестового периода.

Это уменьшает риск информационных утечек и позволяет получить более реалистичную оценку качества модели.

Параметры:

- `n_splits = 3`
- `pctEmbargo = 0.02`

# Управление размером позиции

Вероятность модели используется не только для принятия решения о входе в сделку.

Она также определяет размер позиции.

Чем выше уверенность модели:

- тем больше размер позиции

Тестируются несколько вариантов:

- ML_z_bet
- ML_conf_15
- ML_conf_20
- ML_conf_30

Более высокие значения дают более агрессивную стратегию.

# Backtesting

Финальная проверка выполняется через walk-forward backtesting.

Для каждого нового события:

1. модель обучается только на прошлых данных
2. строится прогноз
3. рассчитывается результат сделки

Такая схема максимально приближена к реальному использованию модели.

# Основные результаты

Исследование показывает:

1. Методы Financial Machine Learning обладают умеренной предсказательной способностью.
2. Meta-labeling позволяет отсеивать часть неудачных momentum-сигналов.
3. ML-модели не всегда увеличивают доходность.
4. ML-подход часто снижает максимальную просадку.
5. Probability-Based Bet Sizing позволяет адаптивно управлять риском.
6. Наиболее полезным применением машинного обучения оказалось управление риском и фильтрация сигналов, а не попытка точного прогнозирования рынка.

# Ограничения исследования

При интерпретации результатов необходимо учитывать несколько ограничений.

1. Анализ проводится на уровне событий, сформированных CUSUM-фильтром, а не на уровне полноценного инвестиционного портфеля. События возникают неравномерно во времени. Финансовые метрики рассчитываются по последовательности событий, а не по непрерывному временному ряду стоимости портфеля. Поэтому полученные значения доходности и риска нельзя трактовать как результаты реальной торговой системы. Показатели доходности, коэффициента Шарпа и максимальной просадки следует рассматривать прежде всего как инструмент сравнения стратегий внутри исследования. 

2. В расчётах не учитываются транзакционные издержки:
   - комиссии
   - bid-ask spread
   - проскальзывание

   На практике эти факторы способны существенно снизить итоговую доходность стратегии.
